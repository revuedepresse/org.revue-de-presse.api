# Symfony 7.4 Migration — Stage 0 (Pre-flight) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land Stage 0 of the Symfony 5.4 → 7.4 LTS migration: commit the in-flight cleanup work, install Rector + PHPStan with extensions, capture a PHPStan baseline at level 5, capture a deprecation baseline for the API endpoints, and resolve the `symfony/uid` Flex recipe drift. End state is a green `phpunit` suite, clean `composer audit`, and a frozen baseline that subsequent stages must not regress.

**Architecture:** This stage adds two static-analysis dev tools (Rector for codemods, PHPStan for type/API analysis) and locks down a baseline. No production code changes. Pre-existing in-session work (Firebase removal, security advisories) is already committed; remaining work is consolidation + tooling install + baseline capture.

**Tech Stack:** Symfony 5.4, PHP ≥ 8.1 (runtime currently 8.4 per xdebug zend_extension path), Doctrine ORM 2, PHPUnit 9.6, Composer Flex.

**Reference spec:** `docs/superpowers/specs/2026-05-06-symfony-7.4-doctrine-3-migration-design.md`


- [ ] **Step 2: Create `phpstan.neon` at project root**

Create file `phpstan.neon`:

```yaml
includes:
    - vendor/phpstan/phpstan-symfony/extension.neon
    - vendor/phpstan/phpstan-symfony/rules.neon
    - vendor/phpstan/phpstan-doctrine/extension.neon
    - vendor/phpstan/phpstan-doctrine/rules.neon
    - phpstan-baseline.neon

parameters:
    level: 5
    paths:
        - src
        - tests
    excludePaths:
        - src/Bluesky/Resources
        - tests/Resources
    symfony:
        containerXmlPath: var/cache/test/App_KernelTestDebugContainer.xml
    doctrine:
        objectManagerLoader: tests/object-manager.php
```

Note: the `containerXmlPath` and `objectManagerLoader` parameters point at files we'll create in the next steps. They're necessary for the Symfony/Doctrine extensions to introspect the container and entity manager.

- [ ] **Step 3: Warm the test cache so the container XML exists**

```bash

```

Expected: cache warm-up completes.

- [ ] **Step 4: Find the actual container XML filename and update `phpstan.neon`**

Symfony names the file `<KernelClass>TestDebugContainer.xml` where `<KernelClass>` depends on the kernel's namespace path. Find it explicitly:

```bash

```

Expected: one path printed, e.g. `var/cache/test/App_KernelTestDebugContainer.xml` or `var/cache/test/srcApp_KernelTestDebugContainer.xml` depending on Symfony version.

Replace the placeholder `containerXmlPath` line in `phpstan.neon` with the actual path returned. Example sed if the file is `App_KernelTestDebugContainer.xml`:

```bash
sed -i.bak 's|containerXmlPath:.*|containerXmlPath: var/cache/test/App_KernelTestDebugContainer.xml|' phpstan.neon
```

- [ ] **Step 3: Decide and document**
