# Symfony 5.4 → 7.4 Migration Implementation Plan (slim codebase)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate `symfony/*` from `^5.4` to `^7.4` in a single PR with internal staging. Doctrine ORM 2 → 3 included. PHP composer platform pin moves from `8.1.6` to a concrete `8.2.x`. `TokenAuthenticator` rewritten against the modern `AbstractAuthenticator` + `Passport` API. End state: `OK (5 tests, 15 assertions)` on Symfony 7.4 + Doctrine ORM 3 + PHP 8.2+.

**Architecture:** Single PR, ~9 internal commits. Rector handles mechanical Symfony 6.x and 7.x deprecation deltas; `DoctrineSetList::ANNOTATIONS_TO_ATTRIBUTES` handles entity attribute conversion. `TokenAuthenticator` is the only significant manual port. PHPStan baseline maintained throughout.

**Tech Stack:** Symfony 5.4 → 7.4, Doctrine ORM 2 → 3, PHP 8.4 runtime (composer pin 8.1.6 → 8.2.x), Rector + rector-symfony + rector-doctrine, PHPStan + phpstan-symfony + phpstan-doctrine.

**Reference spec:** `docs/superpowers/specs/2026-05-07-symfony-7.4-migration-on-slim-codebase-design.md`

- [ ] **Step 1: Read the current TokenAuthenticator + security.yaml**

```bash

cat config/packages/security.yaml

```

Capture the current behavior:
  - `supports()` returns false for `/api/callback` and `/api/healthcheck`, true otherwise
  - `getCredentials()` extracts `x-auth-token` header (or, on OPTIONS preflight, falls back to `getMemberHavingApiKey()` to grab any authenticated apiKey)
  - `getUser()` uses `$userProvider->loadUserByIdentifier($apiKey)` (entity user provider)
  - `checkCredentials()` returns true (token-based, no password)
  - `onAuthenticationSuccess()` returns null (lets request continue)
  - `onAuthenticationFailure()` returns 204 for `/api/healthcheck` (preserved as health-OK), 403 JSON otherwise
  - `start()` returns 401 JSON with "Authentication Required"

- [ ] **Step 2: Rewrite TokenAuthenticator against the new API**

End-state shape of `src/Twitter/Infrastructure/Security/Authentication/TokenAuthenticator.php`:

```php
<?php
declare(strict_types=1);

namespace App\Twitter\Infrastructure\Security\Authentication;

use App\Membership\Domain\Entity\MemberInterface;
use App\Twitter\Infrastructure\DependencyInjection\LoggerTrait;
use App\Twitter\Infrastructure\DependencyInjection\Membership\MemberRepositoryTrait;
use Symfony\Component\HttpFoundation\JsonResponse;
use Symfony\Component\HttpFoundation\Request;
use Symfony\Component\HttpFoundation\Response;
use Symfony\Component\Security\Core\Authentication\Token\TokenInterface;
use Symfony\Component\Security\Core\Exception\AuthenticationException;
use Symfony\Component\Security\Core\Exception\CustomUserMessageAuthenticationException;
use Symfony\Component\Security\Http\Authenticator\AbstractAuthenticator;
use Symfony\Component\Security\Http\Authenticator\Passport\Badge\UserBadge;
use Symfony\Component\Security\Http\Authenticator\Passport\Passport;
use Symfony\Component\Security\Http\Authenticator\Passport\SelfValidatingPassport;
use Symfony\Component\Security\Http\EntryPoint\AuthenticationEntryPointInterface;

class TokenAuthenticator extends AbstractAuthenticator implements AuthenticationEntryPointInterface
{
    use MemberRepositoryTrait;
    use LoggerTrait;

    public function supports(Request $request): ?bool
    {
        $path = $request->getPathInfo();
        if (str_starts_with($path, '/api/callback') || str_starts_with($path, '/api/healthcheck')) {
            return false;
        }

        return true;
    }

    public function authenticate(Request $request): Passport
    {
        if ($request->isMethod('OPTIONS')) {
            $member = $this->memberRepository->getMemberHavingApiKey();
            $apiKey = $member->getApiKey();
        } else {
            $apiKey = $request->headers->get('x-auth-token');
        }

        if ($apiKey === null || $apiKey === '') {
            throw new CustomUserMessageAuthenticationException('No API token provided');
        }

        return new SelfValidatingPassport(new UserBadge($apiKey));
    }

    public function onAuthenticationSuccess(Request $request, TokenInterface $token, string $firewallName): ?Response
    {
        return null;
    }

    public function onAuthenticationFailure(Request $request, AuthenticationException $exception): ?Response
    {
        if ($request->getPathInfo() === '/api/healthcheck') {
            return new Response(
                null,
                Response::HTTP_NO_CONTENT,
                ['status' => 'Everything is ok ^_^']
            );
        }

        return new JsonResponse(
            ['message' => strtr($exception->getMessageKey(), $exception->getMessageData())],
            Response::HTTP_FORBIDDEN
        );
    }

    public function start(Request $request, ?AuthenticationException $authException = null): Response
    {
        return new JsonResponse(['message' => 'Authentication Required'], Response::HTTP_UNAUTHORIZED);
    }
}
```

Behavior preserved:
- Anonymous paths (`/api/callback`, `/api/healthcheck`) skip the authenticator (`supports()` returns false)
- OPTIONS preflight on protected routes uses the first apiKey-having Member (preserved quirk)
- Standard requests use the `x-auth-token` header
- Failure on `/api/healthcheck` returns 204 (preserved health-check semantics)
- Other failures: 403 JSON
- `start()`: 401 JSON

- [ ] **Step 3: Update `config/packages/security.yaml`**

The current firewall config has guard-style:

```yaml
firewalls:
    main:
        anonymous: ~
        logout: ~
        provider: user_provider
        guard:
            authenticators:
                - app.token_authenticator
        stateless: true
```

Symfony 6+ uses:

```yaml
firewalls:
    main:
        provider: user_provider
        custom_authenticators:
            - app.token_authenticator
        entry_point: app.token_authenticator
        stateless: true
```

(Drop `anonymous: ~` and `logout: ~` — both deprecated/removed in 6.0. The access_control rules with `IS_AUTHENTICATED_ANONYMOUSLY` continue to work.)

Apply this to all three `when@<env>:` blocks if the firewall config is duplicated; if it's at the top level (shared across envs), only update once.
