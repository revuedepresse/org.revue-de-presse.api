# Symfony 5.4 → 7.4 LTS migration on the slim codebase — design

**Status:** draft for review
**Date:** 2026-05-07
**Project:** `revuedepresse/http-api`
**Supersedes:** `docs/superpowers/specs/2026-05-06-symfony-7.4-doctrine-3-migration-design.md`
**Predecessor:** the codebase cleanup PR (#390) reduced `src/` from 208 to 25 PHP files.

---

## 1. Why a revised spec

The original spec described a 4-stage migration (5.4 → 6.4 LTS → ORM 3 on 6.4 → 7.4 LTS) over the full 208-file codebase, with each stage as its own PR and explicit gates around an auth flow that had thin test coverage and many transitive dependencies.

After the cleanup PR landed, the calculus changed:
- `src/` is 25 PHP files; the Symfony surface area touched by the keep-set is small.
- `doctrine.yaml` has one annotation mapping group (Membership). Token cluster (restored, not yet mapped) adds 6 entity-class files with `@ORM\*` annotations.
- The auth path now has a regression test (`test_get_highlights_authenticates_with_dummy_token_and_returns_collection_shape`) that exercises real `TokenAuthenticator` flow against a Member fixture.
- `composer.json` is reduced to a tight set: 9 unused deps were dropped.

The intermediate 6.4 LTS stop and the per-stage PR cycle add ceremony that no longer match the risk profile. **One PR with internal staging commits is the right shape.**

## 2. Decisions (resolved during brainstorming)

- **Q1 (hop strategy): direct 5.4 → 7.4.** Single composer bump from `^5.4` to `^7.4`. No intermediate 6.4 stop. Rector covers the deprecation deltas mechanically before the bump.
- **Q2 (Doctrine ORM 2 → 3): same PR.** Sequenced after the Symfony 6.x mechanical deprecation cleanup, before the 7.x jump. ORM 3 supports Symfony 6.4+, so the order works.
- **Q3 (`TokenAuthenticator` port): manual rewrite.** Symfony 6 removed `AbstractGuardAuthenticator`. The current `TokenAuthenticator` is ~140 lines extending the deprecated guard base; it gets rewritten against `Symfony\Component\Security\Http\Authenticator\AbstractAuthenticator` + `Passport`. The auth functional test from the cleanup PR (Task 8) is the regression check.

## 3. Goal and non-goals

### Goal

`composer.json` on `symfony/*: ^7.4`, PHP runtime ≥ 8.2 (composer platform pin updated to a concrete 8.2.x), `doctrine/orm: ^3`, all 25 keep-set PHP files functioning under the new framework, all 5 controller tests green, `composer audit` clean, `bin/console cache:clear` clean across `dev`/`test`/`prod`.

### Non-goals

- Symfony 8.x. Re-evaluate when 8.4 LTS lands (~Nov 2027).
- Test-coverage expansion beyond what the migration strictly requires for verification.
- Architectural changes beyond what the API contract changes force (e.g., security authenticator port).
- Replacing Predis or other still-maintained deps.
- Schema migrations (the cleanup PR established no schema is hydrated by entity hydration in the live paths beyond the Member entity authentication query; the migration must not change the `member` table shape).

## 4. Current state — evidence (post-cleanup)

- All `symfony/*` components on `^5.4`. Composer platform pin `8.1.6`. Runtime PHP 8.4 (per `provisioning/containers/service/templates/extensions.ini.dist` xdebug `zend_extension` path `no-debug-non-zts-20240924`).
- `doctrine/orm: ^2.8`, `doctrine/doctrine-bundle: ^2.2`, `doctrine/annotations: ^2.0`. One mapping group in `doctrine.yaml` (Membership annotation). Token + TokenType + (Member's `tokens` is gone) entities exist as classes with `@ORM\*` annotations, but Token cluster is **not** registered with any mapping group.
- `TokenAuthenticator` extends `AbstractGuardAuthenticator` (deprecated since 5.3, removed in 6.0). Wired in `security.yaml` via `firewalls.main.guard.authenticators: [app.token_authenticator]` style.
- `security.yaml` has three `when@<env>:` blocks (prod, dev, test) — each contains the same `access_control` shape after the cleanup PR. Test env uses real auth (no `firewalls.main.security: false` bypass).
- 5 tests passing on the keep-set. Test setup uses `sqlite:///:memory:` + `SchemaTool::createSchema` + a Member fixture in `setUp()`.
- Previously installed dev tools: none. Rector and PHPStan need installation as part of pre-flight.

## 5. Target state

- `symfony/*: ^7.4` resolved; lock + vendor in sync.
- `doctrine/orm: ^3`, `doctrine/doctrine-bundle: ^2.10` (or whatever ORM 3 demands).
- `doctrine/annotations` removed entirely. All `@ORM\*` annotations on Member + Token + TokenType converted to PHP 8 attributes (`#[ORM\Entity]`, `#[ORM\Column]`, etc.). Doctrine mapping in `doctrine.yaml` flipped from `type: annotation` to `type: attribute`.
- `composer.json` `config.platform.php` from `8.1.6` → `8.2.x` (concrete 8.2 patch, e.g. `8.2.20`). `require.php` from `>=8.1` → `>=8.2`.
- `TokenAuthenticator` rewritten against the modern Symfony Security authenticator interface. `security.yaml` firewall config updated to the new `authenticators:` key.
- Provisioning runtime verified at PHP ≥ 8.2 (the active xdebug path is for PHP 8.4; verify the path matches the installed runtime in the deployed container — likely no actual change needed since runtime is already 8.4+).
- All 5 tests still pass. PHPStan baseline carried forward (no new issues introduced).

## 6. Approach — single PR, internal staging

**Branch strategy:** continue on `http-api` (where the cleanup PR landed). New work as a sequence of small commits, opened as a single PR to `main`.

**Verification gate (continuous, between every internal commit):**

  - `bin/phpunit -c phpunit.xml.dist` reports `OK (5 tests, 15 assertions)`.
  - `bin/console cache:clear --env=dev` and `--env=test` return exit 0.
  - `composer audit` reports no advisories.
  - PHPStan `analyse` reports no new issues vs. baseline.

After the framework bump (Symfony 6.4 → 7.4), additionally:
  - Provisioning PHP version verified ≥ 8.2.
  - PHPStan run on PHP 8.2+.

**Internal stages within the PR:**

1. **Pre-flight tooling.** Install Rector (`rector/rector`, `rector/rector-symfony`, `rector/rector-doctrine`) + PHPStan (`phpstan/phpstan`, `phpstan/phpstan-symfony`, `phpstan/phpstan-doctrine`) as `require-dev`. Create `rector.php` with no rule sets active. Create `phpstan.neon` at level 5 with the Symfony + Doctrine extensions and a generated baseline. Capture a deprecation log against the 5-test run for forensic comparison.

2. **Rector Symfony 6 deprecation cleanup.** Run `SymfonySetList::SYMFONY_60`, `SYMFONY_61`, `SYMFONY_62`, `SYMFONY_63`, `SYMFONY_64` Rector sets sequentially. Each set as its own commit. Diff reviewed before applying. The sets cover: deprecated method renames, container service ID changes, kernel attribute migrations, etc. They do NOT cover `AbstractGuardAuthenticator` (manual change).

3. **Manual `TokenAuthenticator` port + `security.yaml` firewall update.** Rewrite `TokenAuthenticator` against `Symfony\Component\Security\Http\Authenticator\AbstractAuthenticator` and `Passport`. Update `security.yaml`: `firewalls.main.guard.authenticators` → `firewalls.main.custom_authenticators` (or `authenticators`). The `supports()` method, `authenticate()` method (replaces `getCredentials` + `getUser` + `checkCredentials`), `onAuthenticationSuccess` and `onAuthenticationFailure` get refactored. The Task 8 auth test is the regression check.

4. **Composer bump to 6.4.** `composer require 'symfony/*:^6.4' --with-all-dependencies`. Run lock resync, run install. Verify tests + cache:clear.

5. **Doctrine ORM 2 → 3.** Run `DoctrineSetList::ANNOTATIONS_TO_ATTRIBUTES` Rector set against `src/Membership/Domain/Entity/Legacy/Member.php`, the abstract `src/Membership/Domain/Model/Member.php` parent, and the Token cluster (Token, TokenType, TokenInterface, TokenTrait). Diff reviewed entity-by-entity. Update `doctrine.yaml` Membership mapping `type: annotation` → `type: attribute`. Bump `doctrine/orm: ^2.8` → `^3`, drop `doctrine/annotations`. Verify `doctrine:schema:validate --skip-sync` clean. Verify tests + cache:clear.

6. **Rector Symfony 7 deprecation cleanup.** Run `SymfonySetList::SYMFONY_70..74` sequentially. Same review-and-commit pattern as step 2.

7. **Composer bump to 7.4.** `composer require 'symfony/*:^7.4' --with-all-dependencies`. Update `composer.json`: `config.platform.php: '8.1.6'` → `'8.2.x'`, `require.php: '>=8.1'` → `'>=8.2'`. Verify tests + cache:clear on PHP 8.2+ runtime.

8. **Final verification + PR.** Full gate. Open PR (already open as the cleanup PR was merged; either reuse the same branch or open a new one). Manual smoke per Task 10 of the cleanup plan if applicable.

## 7. Risk profile

| Step | Risk | Mitigation |
|---|---|---|
| 1 (tooling) | Very low | Standard composer require + config files; no behavior change |
| 2 (6.x Rector) | Low-medium | Each set committed separately; diff review per set; tests after each |
| 3 (TokenAuthenticator port) | **Medium-high** — concentrated; auth flow is the only meaningful behavior change in the migration | Task 8's auth test (`test_get_highlights_authenticates_with_dummy_token_and_returns_collection_shape` + `test_get_highlights_rejects_unauthenticated_request`) is the gate; passes with old + new code |
| 4 (composer 6.4) | Low — Rector should have handled most deprecations | If composer resolution fails, identify which dep blocks; usually a transitive |
| 5 (ORM 3) | Low — 8 entities to convert via Rector; mechanical | Doctrine schema:validate clean; commit per entity-set if needed |
| 6 (7.x Rector) | Low — most aggressive deprecation cleanup already happened in step 2 | Same diff-review pattern |
| 7 (composer 7.4 + PHP 8.2) | Low — runtime already 8.4 | Provisioning verified ≥ 8.2 before composer bump; xdebug zend_extension path checked |

**Concentrated single-risk:** TokenAuthenticator port (step 3). Everything else is mechanical or low-stakes.

## 8. Rollback model

- All steps live on `http-api`. Each internal stage commits separately. Rollback is `git revert <commit>` for the offending stage.
- No DB schema migrations occur (Member entity columns unchanged through the migration; `doctrine:schema:validate --skip-sync` is the gate that catches drift).
- The cleanup PR's TokenAuthenticator `supports()` narrowing (skip anonymous paths) carries forward into the rewrite — verify the new authenticator preserves that behavior.

## 9. Out-of-scope follow-ups

- Symfony 8.x evaluation (re-visit ~Nov 2027 when 8.4 LTS lands).
- Member entity schema simplification (it has many vestigial columns from the Twitter era; not blocking the migration).
- Test coverage expansion beyond the 5 current tests.
- Restructuring `tests/` setUp into a reusable trait if more tests are added later.

## 10. Open coordination items

- **Provisioning verification.** Confirm the live container's PHP runtime is ≥ 8.2 before the composer 7.4 bump merges. The `xdebug.so` `zend_extension` path in `provisioning/.../extensions.ini.dist` is currently a PHP 8.4 API hash (`no-debug-non-zts-20240924`); if the deployed container actually runs PHP 8.4, no change is needed. If a different PHP version is deployed, the path needs updating.
- **`composer.json` `config.platform.php`** target version. Concrete patch level (e.g., `8.2.20`) tied to the deployed runtime. Confirm the production runtime patch level before pinning.
